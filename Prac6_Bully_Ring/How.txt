Both are used to elect a Leader/Coordinator in a Distributed System. 

Why Election Algorithms are Needed? 
In Distributed Systems, many processes/computers work together. 
Sometimes one special process acts as: 


Coordinator 


Leader 


Master Node 


This leader: 


Manages tasks 


Synchronizes processes 


Controls shared resources 


If leader crashes: 
✅ System must elect a new leader automatically. 
This is solved using: 


Bully Algorithm 


Ring Algorithm 



Real Life Example 
Example: Class Monitor Selection 
Suppose: 


Students have roll numbers. 


Highest roll number becomes monitor. 


If monitor is absent: 


Another student starts election.

1. Bully Algorithm – Viva Explanation 
Aim 
To implement leader election using Bully Algorithm in Distributed Systems. 

Basic Concept 


Every process has unique ID. 


Highest ID process becomes leader. 


If leader crashes: 


Lower process starts election. 


Higher processes reply "OK". 


Highest alive process becomes coordinator. 




Called "Bully" because: 

Highest ID process “bullies” others and becomes leader. 


Real Life Example 
In company: 


Senior-most manager becomes team leader. 


If leader absent, next senior manager takes control. 



Bully Algorithm Flow 
Suppose processes: 
P1 P2 P3 P4 P5 
Leader = P5 
If P5 crashes: 


P2 starts election. 


P2 sends ELECTION to P3,P4,P5 


P3 and P4 reply OK 


P4 starts own election 


Eventually highest alive process wins

Bully Algorithm Viva Questions 
1. What is Bully Algorithm? 
Election algorithm where highest ID alive process becomes coordinator. 

2. Why called Bully? 
Highest priority process forces itself as leader. 

3. Who starts election? 
Any process detecting coordinator failure. 

4. What message types are used? 


ELECTION 


OK 


COORDINATOR 



5. What happens if highest process alive? 
It becomes leader. 

6. Advantage? 
Fast election. 

7. Disadvantage? 
Large number of messages. 

2. Ring Election Algorithm – Viva Explanation 
Aim 
To implement Ring Election Algorithm for electing coordinator in distributed systems. 

Basic Concept 
Processes connected in ring. 
Election message circulates through ring. 
Each process adds its ID. 
Highest ID becomes leader. 

Real Life Example 
People sitting in circle selecting captain. 
Each person says their number. 
Highest number person wins. 

Ring Structure 
P1 → P2 → P3 → P4 → P1 

Ring Algorithm Flow 
Step 1 
P1 starts election. 
Step 2 
Election message moves around ring. 
Step 3 
Each process adds ID. 
Step 4 
When message returns: 
Highest ID selected. 
Step 5 
Leader message circulates.

Ring Algorithm Viva Questions 
1. What is Ring Election Algorithm? 
Election algorithm where processes are arranged in ring topology. 

2. How is leader selected? 
Highest process ID wins. 

3. Why ring topology used? 
Simplifies message passing. 

4. What messages are used? 


ELECTION 


LEADER 



5. Advantage?

6. Disadvantage?
